How It Works

Well, it's pretty simple. Running bijou.exe simply recursively reads the content/ folder applies matching files from the templates/ folder and wries out the result in the site/ folder

  • content/
    • 1.Menu1/ - Folders prefixed with numbers create the navigation hierarchy
      • default.html - Filename that matches a template file and contains the body or data elements.
    • 2.Menu2/ - Ditto
      • default.md - (example of a Markdown template)
    • 3.Menu3/ - Ditto
      • default.csv - (example of a CSV template)
    • 4.Menu4/ - Ditto
      • default.xml - (example of a XML/XSLT template)
    • 0.Invisible/ - Folders prefixed with '0.' are processed by the template engine but are excluded from the navigation hierarchy.
      • default.rss - (example of a RSS feed template).
    • images/ - Folders NOT prefixed with numbers are simply copied (and not processed via templates).
      • logo.png - Files are simply copied.
    • scripts/
      • main.js - Files are simply copied.
    • styles/
      • styles.css - Files are simply copied.
  • templates/
    • default.html - HTML type template (simple substitution of body content)
    • default.csv - CSV type template (converts the CSV file to a table)
    • default.xml - XML type template (applies a XSLT template)
    • default.md - Markdown type template (incomplete Markdown implementation)
    • default.rss - RSS Feed type template (not implemented yet)
    • default.ics - iCalendar type template (not implemented yet)
  • site/
    • The end result. This is where your static HTML site is created.
